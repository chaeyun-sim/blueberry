# 블루베리 프로젝트 개요

## 프로젝트 소개

**Blueberry**는 프리랜서 음악 편곡자를 위한 의뢰/악보/매출 통합 관리 서비스입니다. 편곡 의뢰 수령부터 작업 진행, 완료 및 납품까지의 전체 워크플로우를 하나의 플랫폼에서 관리할 수 있으며, 악보 카탈로그 구축, 매출 통계 분석 등 음악 활동에 필요한 모든 기능을 제공합니다.

**주요 지표:**
- 사용자: 프리랜서 음악 편곡자 및 음악 제작자
- 출시 날짜: 2024년
- 현황: 적극적인 개발 및 성능 최적화 진행 중
- 모바일: PWA 지원 (설치 가능한 웹 앱)

## 타겟 사용자

### Primary User
- **프리랜서 음악 편곡자**: 외주 의뢰를 받아 음악을 편곡하고 수익을 창출하는 개인 사업자
- **음악 제작 관리자**: 프로젝트 진행 상황, 마감일, 의뢰 상태를 체계적으로 추적해야 하는 전문가

### User Needs
- 의뢰 상태 실시간 추적 (수령→작업→완료→납품)
- 악보 파일 중앙 관리 및 빠른 검색
- 월별/연별 수익 분석 및 통계
- 마감일 시각화 및 작업 일정 관리
- 참고 자료(음악 추천) 빠른 접근
- 파일 대량 관리 (ZIP 다운로드)

## 핵심 기능 (7가지)

### 1. 의뢰 관리 (Commission Management)
의뢰의 전체 생명주기를 관리하는 핵심 기능

**주요 기능:**
- 의뢰 등록: 곡 정보, 편곡 스타일, 마감일 입력
- 상태 추적: 수령 → 작업 중 → 완료 → 납품 (4단계 워크플로우)
- AI 이미지 분석: 업로드된 이미지에서 악보/악기 정보 자동 추출
- 의뢰 상세 조회: 입금액, 진행률, 첨부파일, 댓글
- 의뢰 편집: 상태 및 메타데이터 변경

**데이터 모델:**
- commissions (의뢰), arrangements (편곡), categories (악기 분류)

### 2. 악보 관리 (Score Management)
음악 편곡자의 포트폴리오 및 악보 라이브러리 구축

**주요 기능:**
- 악보 등록: 곡명, 편곡 정보, 악기별 파일 업로드
- 악보 상세 조회: 파일 다운로드, 메타데이터 열람
- 악보 검색: 곡명, 편곡자, 악기 분류로 필터링
- 파일 관리: 개별 또는 일괄 다운로드 (ZIP)
- 악보 삭제: Soft Delete (deleted_at 사용)

**데이터 모델:**
- scores (악곡), arrangements (편곡), score_files (악보 파일)

### 3. 매출 통계 (Sales Statistics)
수익 분석 및 사업 관리를 위한 통계 대시보드

**주요 기능:**
- 월별 매출 차트: 추이 시각화 (Recharts)
- 연별 통계: 연도별 총 수익
- Excel 데이터 임포트: XLSX 파일 업로드로 수동 항목 추가
- 통계 조회: 기간별 필터링, 범주별 분류
- 데이터 분석: 최고 수익월, 평균 수익 등 계산

**통합 데이터:**
- 의뢰 입금액 자동 집계
- 수동 매출 항목 추가 가능

### 4. 음악 추천 (Music Recommendation)
영감 제공 및 참고 자료 빠른 접근

**주요 기능:**
- 일일 편곡 추천: 새로운 곡과 편곡 스타일 제시
- Soundpost 레퍼런스: 대한민국 음악 플랫폼의 실시간 핫곡 조회
- 음악 탐색: 장르, 아티스트 기반 추천
- 북마크/저장: 추천 곡 목록 저장

**외부 연동:**
- Soundpost API 통합 (음악 플랫폼 데이터)

### 5. 캘린더 뷰 (Calendar View)
마감일 시각화 및 작업 일정 관리

**주요 기능:**
- 월간 캘린더: 의뢰 마감일 표시
- 일일 상세: 해당 날짜의 의뢰 목록
- 상태별 색상 구분: 수령, 작업, 완료, 납품
- 마감 임박 경고: 마감일 D-day 강조
- 작업 계획 수립: 월간 일정 한눈에 파악

### 6. 파일 관리 (File Management)
악보 및 첨부 파일의 중앙 관리 시스템

**주요 기능:**
- 파일 탐색: 폴더별/악기별 악보 파일 표시
- 검색 및 필터링: 파일명, 곡명, 편곡자로 검색
- 일괄 다운로드: 선택 파일 또는 전체 ZIP으로 다운로드
- 파일 업로드: 드래그&드롭 지원
- 저장소 관리: Supabase Storage 활용

**지원 파일 형식:**
- PDF, MusicXML, MIDI, MP3 등

### 7. 인증 및 설정 (Authentication & Settings)
보안 및 사용자 맞춤 설정

**주요 기능:**
- 이메일/비밀번호 로그인: Supabase Auth 활용
- 게스트 모드: 제한된 기능 접근 (선택 사항)
- 테마 설정: Light/Dark 모드 전환
- 개인 정보 관리: 프로필 수정, 비밀번호 변경
- 알림 설정: 마감일, 상태 변경 푸시 알림 (선택)
- 계정 삭제: 사용자 데이터 일괄 삭제

## 주요 사용 시나리오

### Scenario 1: 의뢰 접수 및 작업 진행
1. 고객으로부터 편곡 의뢰 수신
2. 블루베리에서 새 의뢰 등록 (곡명, 편곡 스타일, 마감일)
3. AI 이미지 분석으로 악기 정보 자동 추출
4. 상태를 "작업 중"으로 변경
5. 작업 완료 시 상태를 "완료"로 변경
6. 납품 후 상태를 "납품"으로 변경
7. 월별 통계에서 해당 의뢰 입금액 자동 집계

**시간 소비:** 의뢰당 5분 (등록 + 상태 변경)

### Scenario 2: 악보 포트폴리오 구축
1. 완성된 악보 파일 수집
2. 악보 관리에서 "새 악보 등록" 클릭
3. 곡명, 편곡자, 악기별 파일 업로드
4. 악보 저장 및 자동 인덱싱
5. 필요시 검색으로 악보 빠르게 조회
6. 클라이언트 요청 시 해당 악보 다운로드 제공

**시간 소비:** 악보당 3분 (메타데이터 입력 + 파일 업로드)

### Scenario 3: 월간 수익 분석
1. 월말에 블루베리 대시보드 접속
2. "매출 통계" 페이지에서 월간 차트 확인
3. 의뢰 입금액이 자동으로 집계됨
4. Excel에서 수동 항목(부가 수익)을 다운로드 템플릿으로 임포트
5. 월간 총 수익, 평균 수익 확인
6. 연별 추이 비교로 사업 성장 분석

**시간 소비:** 월당 10분 (자동 집계 + 수동 임포트)

### Scenario 4: 마감일 기반 작업 관리
1. 월초에 캘린더 뷰 접속
2. 월간 의뢰 마감일을 시각적으로 확인
3. 마감 임박 의뢰 우선순위 결정
4. 일일 캘린더에서 해당 날짜의 의뢰 상세 정보 확인
5. 작업 계획표 작성 (외부 도구와 연동)
6. 마감일 D-day 알림 수신 (선택 사항)

**시간 소비:** 주당 5분 (일정 계획)

### Scenario 5: 파일 대량 관리
1. 클라이언트가 여러 악보 한꺼번에 요청
2. 파일 관리 페이지에서 악기별 필터링
3. 필요한 악보 5개 선택
4. "ZIP 다운로드" 버튼 클릭
5. 클라이언트에게 ZIP 파일 전송

**시간 소비:** 파일 5개당 2분

### Scenario 6: 참고 음악 탐색
1. 새로운 편곡 스타일 찾기 위해 음악 추천 페이지 방문
2. 일일 추천 곡 목록 확인
3. Soundpost에서 최신 핫곡 확인
4. 마음에 드는 곡 재생 및 분석

**시간 소비:** 세션당 15분 (영감 탐색)

## 비즈니스 로직

### 의뢰 워크플로우 (Commission Workflow)

```
┌─────────────┐
│   수령됨     │  (received)
│  Received   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  작업 중     │  (in_progress)
│  In Progress│
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   완료됨     │  (completed)
│  Completed  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   납품됨     │  (delivered)
│ Delivered   │
└─────────────┘
```

**상태 전이:**
- 초기 상태: "수령됨" (의뢰 등록 시)
- 작업 시작: "수령됨" → "작업 중"
- 작업 완료: "작업 중" → "완료됨"
- 최종 납품: "완료됨" → "납품됨"
- 역행 가능: 모든 상태는 이전 상태로 돌아갈 수 있음 (수정/재작업)

**주요 이벤트:**
- 의뢰 생성: 새로운 commission 레코드 생성
- 상태 변경: commission.status 업데이트
- AI 분석: 이미지 업로드 시 Supabase Function 트리거
- 통계 반영: 납품 상태로 변경 시 매출 통계에 자동 포함

### 악보 카탈로그 (Score Catalog)

**구조:**
```
Score (악곡)
├── Arrangement (편곡) - 1:N 관계
│   ├── Composer (작곡가)
│   ├── Category (악기 분류) - M:N 관계
│   ├── ScoreFile (악보 파일) - 1:N 관계
│   │   ├── file_path (Supabase Storage)
│   │   └── file_type (pdf, musicxml, midi, mp3)
│   └── metadata (제작일, 상태, 비고)
└── Tags (검색용 태그)
```

**검색 및 필터링:**
- 곡명 검색: LIKE 쿼리
- 편곡자 필터: arrangement.name
- 악기 필터: categories (다중 선택)
- 생성일 범위: created_at 정렬

**Soft Delete 패턴:**
- deleted_at이 NULL이면 활성
- deleted_at이 SET되면 논리적 삭제 (물리적 삭제 안 함)
- 모든 쿼리에서 `WHERE deleted_at IS NULL` 조건 자동 적용

### 매출 추적 시스템 (Sales Tracking)

**데이터 소스:**
1. **자동 집계**: commissions.amount → monthly_stats
   - 의뢰 상태가 "납품됨"일 때 자동 포함
   - Supabase Realtime으로 실시간 업데이트 (선택)

2. **수동 입력**: manual_sales_entries
   - Excel 임포트로 추가 수익 항목 입력
   - 기타 수입(강의료, 로열티 등) 기록

**통계 계산:**
- 월별 수익: `SUM(commission.amount) WHERE MONTH(due_date) = ?`
- 연별 수익: `SUM(commission.amount) WHERE YEAR(due_date) = ?`
- 평균 수익: `AVG(commission.amount) per month`
- 최고/최저: MAX/MIN 계산

**차트 시각화:**
- Recharts LineChart: 월간 추이
- BarChart: 분기별 비교
- PieChart: 카테고리별 분포 (선택)

### 의뢰 AI 분석 (AI Image Analysis)

**워크플로우:**
1. 사용자가 의뢰 생성 시 이미지 업로드
2. 이미지 → Supabase Storage에 저장
3. `analyze-commission-image` Function 트리거 (Webhook)
4. AI 모델 (Vision API)로 악보 스캔 및 분석
5. 악기 정보 자동 추출 → categories 자동 생성
6. 분석 결과 저장 → commission.analyzed_data

**추출 정보:**
- 악기 목록 (악보에 표시된 악기)
- 곡명 (가능한 경우)
- 박자, 템포 (악보 헤더에서)
- 편곡 스타일 (클래식, 재즈, 팝 등)
- 신뢰도 점수

**사용자 경험:**
- 자동 추출된 정보가 form에 pre-fill
- 사용자가 수정 가능 (AI는 참고용)
- 분석 실패 시 수동 입력 가능
